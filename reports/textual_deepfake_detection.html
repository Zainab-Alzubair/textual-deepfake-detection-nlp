Introduction


Can we predict whether a product review is human-written or AI-generated using textual features?

This project applies Natural Language Processing (NLP) and machine learning to detect AI-generated product reviews. As generative AI makes it easier to produce realistic fake reviews, automated detection has become increasingly important for maintaining trust in online review platforms.

Using a labelled dataset of product reviews, the project compares traditional machine learning models with the transformer-based DistilBERT model to evaluate their effectiveness for textual deepfake detection.
 

1. Problem Statement


Online product reviews strongly influence customer purchasing decisions and trust in e-commerce platforms. However, the widespread availability of generative AI has made it easier to produce realistic fake reviews that can manipulate product ratings, mislead customers, and damage seller reputations.

The aim of this project is to develop a Natural Language Processing (NLP) system that classifies product reviews as either human-written or AI-generated. This is a binary text classification problem with two target classes:

* 0: Human-written / Original Review
* 1: AI-generated / Computer-Generated Review

To address this problem, the project applies text preprocessing, TF-IDF feature extraction, traditional machine learning models, and a transformer-based DistilBERT model to evaluate their effectiveness for AI-generated review detection.
 

2. Business Understanding


The business problem addressed in this project is maintaining trust in online review systems. Product reviews influence customer purchasing decisions, seller reputation, and product visibility, making them valuable targets for AI-generated fake content.

The main stakeholders are:

* Customers, who rely on genuine reviews when making purchasing decisions.
* Sellers, who need fair competition and protection from manipulated ratings.
* E-commerce platforms, which aim to maintain trust and reduce fake content.
* Moderation teams, who require efficient tools to identify suspicious reviews.

The proposed system supports the review moderation process by automatically analysing submitted reviews and flagging those predicted to be AI-generated for human review. Rather than replacing moderators, the model acts as a decision-support tool that helps prioritise suspicious reviews while reducing the risk of incorrectly removing genuine customer feedback.
 

3. High-Level System Design


The proposed system follows an end-to-end NLP classification pipeline that predicts whether a product review is human-written or AI-generated.

The workflow consists of the following stages:

1. Data Collection: Download the labelled fake reviews dataset from Kaggle using KaggleHub.
2. Data Understanding: Explore the dataset, labels, class distribution, and review characteristics.
3. Text Preprocessing: Clean the review text by converting to lowercase and removing URLs, HTML tags, punctuation, numbers, extra spaces, and stop words.
4. Feature Extraction: Convert cleaned reviews into TF-IDF vectors for the traditional machine learning models, while DistilBERT uses tokenised original text.
5. Model Training: Train and compare Logistic Regression, Linear SVM, Multinomial Naive Bayes, and DistilBERT.
6. Model Evaluation: Evaluate performance using accuracy, precision, recall, F1-score, classification reports, and confusion matrices.
7. Deployment: Flag reviews predicted as AI-generated to support human moderation.

This design is appropriate because the problem is a supervised text classification task, allowing the models to learn patterns from labelled reviews and classify unseen reviews.

4.2 Dataset Description


The Fake Reviews Dataset contains 40,432 labelled product reviews for binary text classification. Each review is labelled as either an original human-written review or a computer-generated review.

The original variables are:

* category: Product category
* rating: Product rating
* label: Review class (OR = Original Review, CG = Computer-Generated Review)
* text_: Review text

For modelling, text_ was renamed to review_text, and the labels were encoded as:

* 0: Human-written / Original Review
* 1: AI-generated / Computer-Generated Review

The dataset is balanced, containing 20,216 reviews in each class. After preprocessing, 40,430 reviews remained for modelling.

This dataset is well suited to the project because it provides labelled review text for supervised NLP classification and supports comparison between traditional machine learning models and the DistilBERT transformer model.

8. Text Preprocessing

 

8.1 Text Cleaning Function

In [ ]:
def clean_text(text):

    # Convert to string and lowercase
    text = str(text).lower()

    # Remove URLs
    text = re.sub(r"http\S+|www\S+", "", text)

    # Remove HTML tags
    text = re.sub(r"<.*?>", "", text)

    # Remove punctuation
    text = text.translate(str.maketrans("", "", string.punctuation))

    # Remove numbers
    text = re.sub(r"\d+", "", text)

    # Remove extra spaces
    text = re.sub(r"\s+", " ", text).strip()

    # Remove stop words
    words = text.split()
    words = [word for word in words if word not in ENGLISH_STOP_WORDS]

    return " ".join(words)

9. Feature Extraction using TF-IDF



Machine learning models require numerical input, so the cleaned reviews were converted into TF-IDF (Term Frequency–Inverse Document Frequency) features. TF-IDF assigns greater importance to words that are frequent within a review but less common across the dataset, making it effective for text classification.

Both unigrams (single words) and bigrams (two-word phrases) were used to capture informative words and short expressions. TF-IDF was selected because it is computationally efficient, widely used for NLP tasks, and performs well with linear classifiers such as Linear SVM, providing a strong baseline for comparison with DistilBERT.

12.7 Define Evaluation Metrics

In [ ]:
def compute_metrics(eval_pred):
    predictions, labels = eval_pred
    predictions = np.argmax(predictions, axis=1)

    accuracy = accuracy_score(labels, predictions)
    precision = precision_score(labels, predictions)
    recall = recall_score(labels, predictions)
    f1 = f1_score(labels, predictions)

    return {
        "accuracy": accuracy,
        "precision": precision,
        "recall": recall,
        "f1": f1
    }

13.2 Comparative Analysis


Linear SVM was the best-performing traditional model, achieving 91.02% accuracy and an F1-score of 90.97%. DistilBERT achieved the highest overall performance, with 95.75% accuracy and an F1-score of 95.85%, outperforming all traditional models.

These results show that transformer-based models provide better detection performance for AI-generated reviews, while Linear SVM remains a strong alternative due to its lower computational cost and faster deployment. Therefore, DistilBERT is recommended when maximum accuracy is required, whereas Linear SVM is more suitable for efficient large-scale applications.
 

14. Critical Evaluation


DistilBERT achieved the highest overall performance, with 95.75% accuracy and an F1-score of 95.85%, while Linear SVM was the strongest traditional model, achieving 91.02% accuracy and an F1-score of 90.97%. These results show that DistilBERT outperformed the traditional TF-IDF models.

However, Linear SVM remains a practical alternative because it provides strong performance with lower computational requirements and simpler deployment. Since all models were trained and evaluated using the same dataset split, the comparison is both fair and reliable.
 

15. Deployment Considerations


The proposed system could be integrated into an e-commerce moderation workflow, where suspicious reviews are flagged for human review. DistilBERT provides the highest accuracy, while Linear SVM offers faster large-scale deployment. A hybrid approach combines both advantages, and regular retraining helps maintain performance.
 

16. Limitations and Future Work


The dataset represents reviews from a single source and may not generalise to other domains or languages. DistilBERT also requires greater computational resources than traditional machine learning models. Future work could evaluate larger transformer models, incorporate reviewer metadata and behavioural features, and implement continuous retraining to adapt to evolving AI-generated content.
 

17. Conclusion


This project developed an end-to-end NLP pipeline for detecting AI-generated product reviews. Among the traditional machine learning models, Linear SVM achieved the best performance, while DistilBERT produced the highest overall accuracy (95.75%). The results demonstrate that transformer-based models provide excellent detection performance, while traditional approaches remain suitable for efficient large-scale deployment.
